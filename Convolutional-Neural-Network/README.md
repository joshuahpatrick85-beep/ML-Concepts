# Convolutional Neural Network (CIFAR-10)

## Goal
Build and train a CNN using PyTorch's `nn.Conv2d`/pooling layers on CIFAR-10, and compare its performance against a fully-connected (FFNN) baseline on the same dataset.

## Approach
- Loaded CIFAR-10 via `torchvision.datasets.CIFAR10`, normalized, and batched with `DataLoader` (batch size 32).
- Established an FFNN baseline: `Flatten → Linear(3072,512) → ReLU → Linear(512,128) → ReLU → Linear(128,10)`, trained 5 epochs with SGD (lr=0.01) and cross-entropy loss.
- Built a CNN: three `Conv2d(kernel=5, padding=2) → ReLU → MaxPool2d(2)` blocks (channels 3→16→32→64), followed by `Flatten → Linear(1024,10)`. Padding was chosen so each conv layer preserves spatial size, with each pooling layer halving it; three pooling steps bring 32×32 down to 4×4, giving a flattened size of 4×4×64 = 1024 into the final linear layer. Trained 10 epochs with the same optimizer and loss setup as the FFNN.
- Evaluated both models on the held-out test set (no gradient updates) and compared accuracy between the two.

## Results
- FFNN baseline test accuracy: 50.42%
- CNN test accuracy: 70.19%
